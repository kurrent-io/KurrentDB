# Testing Scripts

Scripts for running tests and generating reports in the Kurrent DB.

## test-runner.cs

Canonical test runner for the solution. Wraps `dotnet test` with category filtering, unique result folders,
live status output, and automatic report generation.

```bash
dotnet scripts/testing/test-runner.cs -- run unit                                     # all unit tests
dotnet scripts/testing/test-runner.cs -- run integration                              # all integration tests
dotnet scripts/testing/test-runner.cs -- run all                                      # everything
dotnet scripts/testing/test-runner.cs -- run unit --treenode-filter "/*/*/MyTests/*"  # specific class
dotnet scripts/testing/test-runner.cs -- run unit --run-id "$(uuidgen)"               # fixed (unique) results folder
dotnet scripts/testing/test-runner.cs -- list unit                                    # list without running
```

Each run creates a folder under `.artifacts/test-results/<run-id>/` containing:

- `*.trx` — one TRX file per test assembly, renamed to `{assembly}-{os}-{tfm}-report.trx`
- `*.md` — one markdown report per assembly, generated by `trx2md.py`
- `*.html` — one HTML report per assembly, generated by TUnit (only when `--html-report` is passed)
- `output.log` — raw `dotnet test` output, last resort for debugging runner/infrastructure issues

The folder name is auto-generated (`<timestamp>-<4hex>`) unless you pass `--run-id`. Supply your own id when
you need the report path upfront — read `.artifacts/test-results/<run-id>/*.md` directly instead of scraping
the console for the generated folder. The id **must be unique per run** (use a GUID); reusing a value writes
into the same folder and mixes reports.

### Options

| Flag                | Description                                                                 |
|---------------------|----------------------------------------------------------------------------|
| `--treenode-filter` | TUnit treenode filter pattern                                              |
| `--html-report`     | Generate TUnit HTML reports (disabled by default)                          |
| `--rebuild`         | Force a full from-scratch build (`--no-incremental`) before running tests  |
| `--run-id`          | Use this fixed, **unique** results folder name — a GUID (`.artifacts/test-results/<run-id>`) — instead of an auto-generated one |

The runner builds first (`dotnet build -c Release`), then runs the tests with `--no-build`, so the build
happens exactly once. Pass `--rebuild` to switch that build to `--no-incremental` — needed when you want
the trim/AOT analyzers to re-run from scratch, since an up-to-date incremental build skips compilation and
emits no analyzer warnings.

### Exit codes

`dotnet test`'s raw aggregate exit is unreliable in this mixed-framework solution: non-TUnit assemblies
(xUnit/NUnit on MTP) reject the TUnit-only `--treenode-filter` with exit 5 even when every matched test
passed. The runner therefore computes its exit from the parsed summary:

| Exit | Meaning                                                        |
|------|----------------------------------------------------------------|
| 0    | All matched tests passed                                       |
| 2    | At least one test (or assembly) failed                         |
| 8    | The filter matched zero tests across all assemblies            |
| else | Unexpected `dotnet test` exit (crash/infra), surfaced verbatim |

### CI rsp opt-out

`src/Directory.Build.props` injects `ci/ci.rsp` (`--coverage`, hangdump flags, `--report-trx`) into
every test project for CI. The runner sets `DisableCiTestRsp=true` on **all** its invocations to skip
that injection: coverage/hangdump are CI-only concerns (and `.coverage` files would litter the results
folders), the runner passes `--report-trx` itself, and the rsp's `--report-trx` outright breaks TUnit's
`--list-tests` discovery. CI invokes `dotnet test` directly (variable unset) and is unaffected.

### `list` caveats

- TUnit (observed on 1.0.39) ignores `--treenode-filter` during discovery, so `list` shows **all**
  discovered tests in all TUnit assemblies regardless of filter. Non-TUnit assemblies don't support the
  flag at all; their per-assembly usage-dump noise is suppressed from the output.

Extra TUnit args pass through after the category:

```bash
dotnet scripts/testing/test-runner.cs -- run integration --timeout 30s --maximum-parallel-tests 1
```

## aot-report.cs

Generates an AOT/trim compatibility report. C# file-based app, same stack as `test-runner.cs`. It
discovers every project marked `<IsAotCompatible>true</IsAotCompatible>`, builds each with
`--no-incremental` (so the trim/AOT analyzers re-run from scratch), parses the `IL####` warnings, and
writes a markdown report grouped by project.

```bash
dotnet scripts/testing/aot-report.cs                        # -> .artifacts/aot-report.md
dotnet scripts/testing/aot-report.cs -- --stdout            # print to stdout instead of writing a file
dotnet scripts/testing/aot-report.cs -- --output report.md  # write to a specific path
```

Warnings are split into **own code** (actionable `IL2026`/`IL3050`/`IL2070`/… in our source) and
**packages** (`IL3058` — dependencies not marked AOT-compatible, listed with their resolved versions).
The title is derived from the solution file name; known false positives (e.g. compile-only
`JetBrains.Annotations`) are filtered out; the code legend sits at the end.

### Options

| Flag       | Description                                                  |
|------------|-------------------------------------------------------------|
| `--output` | Markdown output path (default: `.artifacts/aot-report.md`)  |
| `--stdout` | Print the report to stdout instead of writing a file        |

## trx2md.py

Generates markdown test reports from TRX files. Standalone CLI tool — used by `test-runner.cs`
automatically, but can also be run directly.

```bash
# Default: one .md per .trx in current directory (default preset)
./scripts/testing/trx2md.py

# Process specific files
./scripts/testing/trx2md.py path/to/*.trx

# Merged report to stdout (for terminal display)
./scripts/testing/trx2md.py --stdout --preset ai *.trx

# Merged report to a single file
./scripts/testing/trx2md.py --output test-report.md *.trx

# With subtitle and extra metadata
./scripts/testing/trx2md.py --subtitle "integration tests" --extra "net10.0" *.trx
```

### Presets

| Preset    | Description                                                         |
|-----------|---------------------------------------------------------------------|
| `default` | Failed + skipped, full stacks, metadata, grouped by namespace/class |
| `ai`      | Compact, no emoji, inline class names, compressed stacks            |
| `ci`      | Failures only, minimal, top 5 slowest                               |
| `debug`   | Everything including passed, full stacks, no root stripping         |

### Options

```
Sections:   --failed/--no-failed  --skipped/--no-skipped  --passed/--no-passed
Header:     --header-style banner|table|compact  --emoji/--no-emoji  --metadata
Grouping:   --group-by-namespace  --class-name inline|group|hidden
            --categories  --skip-reasons
Stack:      --compress-stack/--no-compress-stack  --max-frames N
            --strip-root/--no-strip-root  --repo-root PATH
Extra:      --slowest N  --subtitle TEXT  --extra VALUE (repeatable)
Config:     --config FILE (JSON)
```

### Config File

```json
{
  "preset": "default",
  "sections": { "failed": true, "skipped": true, "passed": false },
  "header": { "style": "banner", "emoji": true, "metadata": true },
  "grouping": { "namespace": true, "class_name": "group", "categories": true, "skip_reasons": true },
  "stack": { "compress": false, "max_frames": 20, "strip_root": true },
  "slowest": 0
}
```
